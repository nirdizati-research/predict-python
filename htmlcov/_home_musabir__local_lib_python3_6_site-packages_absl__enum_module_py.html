15"""Import workaround so that Bazel, Py2/Py3, and enum34 package work together. 

16 

17This works around a problem due to the combination of Bazel putting 

18third party packages before the stdlib in PYTHONPATH. What happens is: 

19 * The enum34 PyPi package is imported as 'enum'. 

20 * Bazel puts the path to enum34 before the stdlib, hence 'import enum' 

21 will prefer to use enum34 from above instead of the stdlib. 

22 * In Python 3, enum34 is used instead of the stdlib, which breaks 

23 lots of things. It works fine in Python 2, since there is no enum 

24 module. 

25 

26To work around this, we do 3 things: 

27 1. Put the enum34 code on PYTHONPATH, but not directly importable as 

28 'enum'; it is under the (non importable) directory name with the 

29 PyPi package name and version. 

30 2. Try to import enum normally, if it works, great. This makes Py3 work 

31 (as well as Py2 when enum is available as normal). 

32 3. If the normal enum import failed, then try to find the enum34 

33 entry on sys.path, and append the missing directory name. 

34 

35Once it is successfully imported, expose the module directly. This 

36prevents importing the module twice under different names. e.g., 

37the following is true: 

38 from absl._enum_module import enum as absl_enum 

39 import enum as normal_enum 

40 assert absl_enum is normal_enum 

41""" 

42# pylint: disable=unused-import 

43import sys 

44import six 

45 

46try: 

47 import enum 

48except ImportError: 

49 if six.PY3: 

50 # While not all Py3's have enum, only the ones we support do. 

51 raise 

52 for i, path in enumerate(sys.path): 

53 if '/enum34_archive' in path: 

54 sys.path[i] = path + '/enum34-1.1.6' 

55 

56 import enum
